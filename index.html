What is a PLC?

Programmable Logic Controllers are industrial computers...

A PLC is a ruggedized computer used for industrial automation. It continuously monitors input devices (sensors, switches) and makes decisions based on a custom program to control output devices (motors, lights, valves).

üí° PLCs are designed for harsh industrial environments with extreme temperatures, vibration, and electrical noise.

PLC Components

Essential parts of every PLC system...

CPU: Executes the control program
Input Modules: Interface with sensors and switches
Output Modules: Control actuators and indicators
Power Supply: Provides regulated power
Programming Device: Computer or HMI for programming

PLC Scan Cycle

How PLCs execute programs...

The PLC operates in a continuous loop:

1. READ INPUTS ‚Üí Store in memory 2. EXECUTE PROGRAM ‚Üí Process logic 3. UPDATE OUTPUTS ‚Üí Send to devices 4. HOUSEKEEPING ‚Üí Check system 5. REPEAT...
‚ö†Ô∏è Scan time typically ranges from 1-20ms depending on program complexity.

Ladder Logic Fundamentals

The most common PLC programming language...

Ladder Logic resembles electrical relay circuits with rungs. Each rung represents a control statement with inputs (contacts) on the left and outputs (coils) on the right.

|--[ ]--[ ]--( )--| Normally Open contacts |--[/]--[/]--( )--| Normally Closed contacts |--[P]------------| Positive edge trigger

Basic Contacts

Input instructions in ladder logic...

XIC (Examine If Closed): --[ ]-- Passes power when input is ON
XIO (Examine If Open): --[/]-- Passes power when input is OFF
OSR (One Shot Rising): Triggers once on rising edge
üí° Contacts can represent physical inputs or internal memory bits.

Output Instructions

Controlling outputs and memory...

OTE (Output Energize): --( )-- Standard output coil
OTL (Output Latch): --(L)-- Sets and holds ON
OTU (Output Unlatch): --(U)-- Resets latched output
Example Start/Stop Circuit: |--[START]--[STOP]---(RUN)--| |--[RUN]----[/STOP]---------|

Series vs Parallel

Logic combinations...

SERIES (AND Logic): |--[A]--[B]--( OUT )--| PARALLEL (OR Logic): |--[A]--------( OUT )--| |--[B]-----------------|

Series = ALL conditions must be true. Parallel = ANY condition can be true.

PID Control

Process control fundamentals...

PID = Proportional + Integral + Derivative control for maintaining setpoints.

PID Formula: Output = Kp*Error + Ki*‚à´Error*dt + Kd*(dError/dt) Kp = Proportional Gain Ki = Integral Gain Kd = Derivative Gain
P: Immediate response to current error
I: Eliminates steady-state error over time
D: Dampens oscillations, predicts future error
üí° Start tuning with P only, then add I, finally add D if needed.

PID Tuning Methods

Getting optimal control performance...

Manual Tuning: Adjust Kp until stable oscillation, then add Ki and Kd
Ziegler-Nichols: Mathematical method using ultimate gain
Auto-Tune: Built-in PLC function for automatic tuning
‚ö†Ô∏è Never tune PID on a live production process - use simulation or off-hours.
